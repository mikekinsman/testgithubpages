📁 Docs Folder Deployment Example

This page is served from the /docs folder!

This demonstrates the "Deploy from /docs" GitHub Pages configuration option.

Why Use /docs Folder?

  • Separation: Keep your documentation separate from source code
  • Organization: Clear structure for projects with both code and docs
  • Simple Setup: No need for separate branches or build processes
  • Version Control: Documentation stays in sync with code versions

Configuration Steps

  1. Create a /docs folder in your repository root
  2. Add your HTML files, starting with index.html
  3. Go to repository Settings → Pages
  4. Select "Deploy from a branch"
  5. Choose "main" branch and "/docs" folder
  6. Save and wait for deployment

Directory Structure

repository-root/ ├── src/ # Your application source code │ ├── index.js │ └── app.js ├── docs/ # GitHub Pages content (this folder!) │ ├── index.html # Landing page │ ├── guide.html # Documentation pages │ └── assets/ # CSS, images, etc. │ └── style.css ├── README.md # Repository readme └── _config.yml # Jekyll config (optional)

Best Practices

  • Always include an index.html as the entry point
  • Use relative paths for links within docs folder
  • Consider adding a 404.html for custom error pages
  • Organize assets in subfolders (css/, js/, images/)
  • Keep documentation in sync with code changes
Pro Tip: You can use both static HTML (like this page) and Jekyll templates in the /docs folder. GitHub Pages will process Jekyll files automatically!
