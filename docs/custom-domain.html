🌐 Custom Domain Configuration

Set up a custom domain for your GitHub Pages site

Overview

By default, GitHub Pages sites are available at username.github.io or username.github.io/repository. However, you can configure a custom domain like www.example.com or example.com.

Step-by-Step Configuration

Step 1: Choose Your Domain Type

Decide which type of domain you want to use:

  • Apex Domain: example.com (no www)
  • Subdomain: www.example.com or blog.example.com
  • Both: Configure both with redirect
Recommendation: Use www subdomain for better flexibility and to avoid potential DNS issues.

Step 2: Add CNAME File to Repository

Create a file named CNAME (all caps, no extension) in your repository root or docs folder (depending on your source):

# For www subdomain
www.example.com

# OR for apex domain
example.com
Important: The CNAME file should contain only your domain name, nothing else!

Step 3: Configure DNS Records

Log into your domain registrar's DNS management panel and add the appropriate records:

For Apex Domain (example.com):

Type Name Value
A @ 185.199.108.153
A @ 185.199.109.153
A @ 185.199.110.153
A @ 185.199.111.153

For www Subdomain:

Type Name Value
CNAME www username.github.io

For Custom Subdomain (e.g., blog.example.com):

Type Name Value
CNAME blog username.github.io

Step 4: Configure in GitHub Settings

  1. Go to your repository settings
  2. Navigate to "Pages" section
  3. Under "Custom domain", enter your domain
  4. Click "Save"
  5. Wait for DNS check to complete (may take a few minutes)

Step 5: Enable HTTPS

After DNS verification completes:

  1. Check "Enforce HTTPS" in Pages settings
  2. Wait for SSL certificate provisioning (can take up to 24 hours)
  3. Your site will be available over HTTPS
Note: GitHub provides free SSL certificates via Let's Encrypt!

Troubleshooting DNS

Verify DNS Configuration

Use command-line tools to verify your DNS setup:

# Check A records for apex domain
dig example.com +noall +answer

# Check CNAME record for subdomain
dig www.example.com +noall +answer

# Alternative (Windows)
nslookup example.com
nslookup www.example.com

Common DNS Configurations

Scenario DNS Setup CNAME File
Only www subdomain CNAME: www → username.github.io www.example.com
Only apex domain 4 A records → GitHub IPs example.com
Both (apex redirects to www) 4 A records + CNAME for www www.example.com
Custom subdomain CNAME: subdomain → username.github.io subdomain.example.com

Update Jekyll Configuration

After setting up your custom domain, update _config.yml:

# _config.yml
url: "https://www.example.com"  # your custom domain with protocol
baseurl: ""  # empty for custom domains

# Optional: enforce custom domain
enforce_ssl: www.example.com

Common Issues and Solutions

Issue: "Domain is improperly configured"
Solution: Wait 24-48 hours for DNS propagation, or check that DNS records are correctly configured.
Issue: HTTPS not available
Solution: Ensure DNS is fully propagated (use dig/nslookup), then wait up to 24 hours for certificate provisioning.
Issue: Site shows 404 error
Solution: Verify CNAME file is in the correct location (root or docs folder based on your Pages source setting).
Issue: Old domain still shows
Solution: Clear browser cache or test in incognito mode. DNS changes can be cached.

Best Practices

  • Always use HTTPS for security and SEO benefits
  • Set up both apex and www subdomain, redirect one to the other
  • Use DNS provider with fast propagation (Cloudflare, DNS Made Easy)
  • Monitor SSL certificate expiration (GitHub auto-renews)
  • Test site on multiple devices after DNS changes
  • Keep CNAME file in version control
Pro Tip: Using Cloudflare? You can use their proxy (orange cloud) for additional security and CDN benefits, but it may require additional configuration for SSL.
