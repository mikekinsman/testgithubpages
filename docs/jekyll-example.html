🎨 Jekyll on GitHub Pages

Understanding Jekyll configuration and its impact on your GitHub Pages site

What is Jekyll?

Jekyll is a static site generator that's built into GitHub Pages. It transforms plain text into static websites and blogs without databases, allowing you to use templates, layouts, and dynamic content generation.

Key Benefit: Write content in Markdown, use layouts and includes, and let Jekyll generate the final HTML automatically.

_config.yml Configuration

The _config.yml file is the heart of Jekyll configuration. Here are key options:

# Basic Site Settings
title: Your Site Name
description: A brief description
author: Your Name
email: your-email@example.com

# Theme Configuration
theme: minima
# or use remote theme:
# remote_theme: owner/theme-repo

# URL Settings
baseurl: "/project-name"  # for project pages
url: "https://username.github.io"

# Build Settings
markdown: kramdown
highlighter: rouge

# Plugins (GitHub Pages supports these)
plugins:
  - jekyll-feed          # RSS feed
  - jekyll-seo-tag       # SEO optimization
  - jekyll-sitemap       # Sitemap generation
  - jekyll-mentions      # @mentions
  - jekyll-redirect-from # Redirect support
  - jekyll-paginate      # Pagination

# Permalink Structure
permalink: /:categories/:year/:month/:day/:title/
# or: pretty, date, ordinal, none

Front Matter

Jekyll uses YAML front matter to define metadata for pages and posts:

---
layout: post
title: "My Blog Post"
date: 2025-01-15 10:30:00 -0800
categories: tutorial jekyll
tags: [github-pages, static-site]
author: Mike
description: "A comprehensive guide to Jekyll"
---

Liquid Templating

Jekyll uses Liquid for templating. Common patterns:

<!-- Variables -->
{{ page.title }}
{{ site.author }}
{{ content }}

<!-- Conditionals -->
{% if page.featured %}
  <div class="featured">Featured Post</div>
{% endif %}

<!-- Loops -->
{% for post in site.posts %}
  <h2>{{ post.title }}</h2>
{% endfor %}

<!-- Includes -->
{% include header.html %}
{% include footer.html %}

Directory Structure

repository/
├── _config.yml          # Jekyll configuration
├── _layouts/            # Page templates
│   ├── default.html
│   └── post.html
├── _includes/           # Reusable components
│   ├── header.html
│   └── footer.html
├── _posts/              # Blog posts
│   └── 2025-01-15-my-post.md
├── _sass/               # Sass stylesheets
│   └── main.scss
├── assets/              # Static files
│   ├── css/
│   └── images/
└── index.md             # Homepage

Supported Themes

GitHub Pages supports several official themes without additional setup:

  • minima - Clean and simple default theme
  • jekyll-theme-cayman - Project page theme with prominent header
  • jekyll-theme-slate - Dark theme with slate background
  • jekyll-theme-architect - Architectural header design
  • jekyll-theme-minimal - Minimal sidebar theme
  • jekyll-theme-dinky - Compact with left sidebar
Note: You can also use any remote theme from GitHub by using remote_theme: username/repo in your _config.yml.

Jekyll vs Plain HTML

Feature Plain HTML Jekyll
Setup Complexity Very Simple Moderate
Content Format HTML Markdown/HTML
Templating Manual Automatic
Blogging Support Manual Built-in
Themes DIY Ready-made

Local Development

Test your Jekyll site locally before pushing:

# Install Jekyll
gem install bundler jekyll

# Create Gemfile
source 'https://rubygems.org'
gem 'github-pages', group: :jekyll_plugins

# Install dependencies
bundle install

# Serve locally
bundle exec jekyll serve

# View at http://localhost:4000
