GitHub Pages Configuration Options Demo

A comprehensive guide to GitHub Pages configuration and deployment options

🚀 Deployment Source Options

GitHub Pages can be deployed from different sources. Each has different use cases:

1. Deploy from Root Directory (main branch /)

Content is served directly from the root of your main branch.

Best for: Simple single-page sites, landing pages, project documentation

Configuration: Settings → Pages → Source: Deploy from branch → Branch: main → Folder: / (root)

Repository Structure:
/
├── index.html
├── about.html
├── style.css
└── README.md

2. Deploy from /docs Directory (main branch /docs)

Content is served from the /docs folder, keeping source code separate from documentation.

Best for: Projects with both code and documentation, organized documentation sites

Configuration: Settings → Pages → Source: Deploy from branch → Branch: main → Folder: /docs

Repository Structure:
/
├── src/           (your application code)
├── docs/          (GitHub Pages content)
│   ├── index.html
│   └── assets/
└── README.md

3. Deploy from gh-pages Branch

Content is served from a dedicated gh-pages branch, completely separate from your main codebase.

Best for: Generated documentation, build artifacts, keeping deployment separate from development

Configuration: Settings → Pages → Source: Deploy from branch → Branch: gh-pages → Folder: / (root)

# Create and use gh-pages branch
git checkout --orphan gh-pages
git reset --hard
# Add your content
git add .
git commit -m "Initial GitHub Pages commit"
git push origin gh-pages

4. GitHub Actions Deployment

Use custom workflows to build and deploy your site with full control over the build process.

Best for: Complex build processes, multiple frameworks, custom deployment logic

Configuration: Settings → Pages → Source: GitHub Actions

# .github/workflows/deploy.yml
name: Deploy GitHub Pages
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Build
        run: npm run build
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3

🎨 Jekyll Configuration Options

GitHub Pages natively supports Jekyll. Configure it using _config.yml:

Basic Jekyll Configuration

# _config.yml
title: Your Site Title
description: Your site description
theme: minima
baseurl: ""  # subpath of your site, e.g., /blog
url: "https://username.github.io"  # base hostname

# Build settings
markdown: kramdown
plugins:
  - jekyll-feed
  - jekyll-seo-tag

# Exclude files from processing
exclude:
  - README.md
  - Gemfile
  - Gemfile.lock

Supported Jekyll Themes

GitHub Pages supports several built-in themes:

  • minima - Default theme, clean and simple
  • jekyll-theme-cayman - Project page theme
  • jekyll-theme-slate - Dark theme
  • jekyll-theme-architect - Header-focused theme
  • jekyll-theme-dinky - Sidebar theme
  • jekyll-theme-minimal - Minimal theme
Note: To use a theme, add it to _config.yml: theme: jekyll-theme-cayman

🌐 Custom Domain Configuration

Setting Up a Custom Domain

  1. Add a CNAME file to your repository root with your domain
  2. Configure DNS records at your domain provider
  3. Enable HTTPS in repository settings
# CNAME file
www.example.com

# DNS Configuration (at your domain provider)
# For apex domain (example.com):
A     @     185.199.108.153
A     @     185.199.109.153
A     @     185.199.110.153
A     @     185.199.111.153

# For www subdomain:
CNAME www   username.github.io

⚙️ Advanced Configuration

Base URL Configuration

When deploying to a project page (username.github.io/projectname), configure the base URL:

# _config.yml
baseurl: "/projectname"
url: "https://username.github.io"

# In HTML/Markdown, reference assets:
{{ site.baseurl }}/assets/image.png
{{ site.url }}{{ site.baseurl }}/page.html

Permalink Configuration

Control how URLs are structured for blog posts and pages:

# _config.yml
permalink: /:categories/:year/:month/:day/:title:output_ext
# or
permalink: /blog/:title/
# or
permalink: pretty  # /category/year/month/day/title/

Environment Variables

Jekyll provides environment-specific configuration:

# Different settings for development vs production
# _config.yml (production)
url: "https://username.github.io"

# _config_dev.yml (development)
url: "http://localhost:4000"

# Run locally:
jekyll serve --config _config.yml,_config_dev.yml

📦 Static Site Generators

Supported Options Beyond Jekyll

With GitHub Actions, you can use any static site generator:

  • Hugo - Fast Go-based generator
  • Next.js - React framework with static export
  • Gatsby - React-based static site generator
  • VuePress - Vue-powered static site generator
  • Docusaurus - Documentation-focused generator
  • Eleventy (11ty) - Simpler JavaScript alternative
Tip: Use GitHub Actions to build these frameworks and deploy to GitHub Pages

🔍 Common Configuration Scenarios

Scenario 1: Simple Project Documentation

Use: /docs folder with Jekyll

Why: Keeps docs with code, easy to maintain, no build step needed

Scenario 2: React/Vue Application

Use: GitHub Actions with build step

Why: Requires build process, can optimize for production

Scenario 3: Personal Blog or Portfolio

Use: Root directory with Jekyll theme

Why: Simple setup, built-in theming, blogging features

Scenario 4: Complex Multi-Environment Site

Use: gh-pages branch with GitHub Actions

Why: Separation of concerns, automated deployments, environment-specific builds
