# Contributing to GitHub Pages Configuration Demo

Thank you for your interest in contributing to this demonstration repository! This project aims to help developers understand GitHub Pages configuration options.

## How to Contribute

### Reporting Issues
If you find errors in the documentation or have suggestions for improvements:
1. Check if an issue already exists
2. Create a new issue with a clear description
3. Include specific examples or screenshots if relevant

### Suggesting Enhancements
We welcome suggestions for:
- Additional configuration examples
- Clearer documentation
- New demonstration pages
- Better code samples
- Improved styling

### Submitting Pull Requests

1. **Fork the repository**
   ```bash
   git clone https://github.com/yourusername/testgithubpages.git
   cd testgithubpages
   ```

2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Keep changes focused and minimal
   - Maintain consistent formatting
   - Test locally before committing

4. **Test your changes**
   - For Jekyll changes: `bundle exec jekyll serve`
   - For HTML changes: Use a local server to preview
   - Verify all links work correctly

5. **Commit your changes**
   ```bash
   git add .
   git commit -m "Description of your changes"
   ```

6. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

7. **Open a Pull Request**
   - Provide a clear description of your changes
   - Reference any related issues
   - Explain the benefit of your contribution

## Guidelines

### Documentation
- Use clear, concise language
- Include code examples where appropriate
- Provide screenshots for visual changes
- Keep explanations accessible to beginners

### Code Style
- Use semantic HTML5 elements
- Keep CSS organized and commented
- Follow existing naming conventions
- Ensure responsive design

### HTML
- Use proper indentation (2 or 4 spaces)
- Include proper meta tags
- Validate HTML structure
- Test in multiple browsers

### Jekyll/Markdown
- Follow Jekyll conventions
- Use proper YAML front matter
- Test Liquid template syntax
- Validate Markdown formatting

### Configuration Files
- Add comments for complex settings
- Maintain YAML/JSON formatting
- Document any non-standard configurations

## What to Contribute

### High Priority
- Additional configuration examples
- Missing deployment scenarios
- Common troubleshooting solutions
- Better visual examples

### Medium Priority
- Improved styling
- Additional theme examples
- More comprehensive error handling
- Performance optimizations

### Nice to Have
- Interactive demos
- Video tutorials
- Multi-language support
- Advanced Jekyll features

## Testing Checklist

Before submitting a PR, ensure:
- [ ] All HTML files have valid syntax
- [ ] Links between pages work correctly
- [ ] Responsive design works on mobile
- [ ] No browser console errors
- [ ] Jekyll builds without errors (if applicable)
- [ ] Code follows repository style
- [ ] Documentation is updated

## Code of Conduct

This is a demonstration and educational project. Please:
- Be respectful and constructive
- Focus on improving the documentation
- Help others learn about GitHub Pages
- Keep discussions professional

## Questions?

If you have questions about contributing:
1. Check existing issues and PRs
2. Review the README.md and SETUP.md
3. Open an issue for discussion
4. Reach out to repository maintainers
